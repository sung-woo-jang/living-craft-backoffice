- @.claude/script/show_diff.sh 하고 @.claude/script/show_recent_commits.sh 의 내용으로 아래의 규칙을 따라 커밋메시지를 만들어

```markdown
- 대답은 항상 한국말로 존댓말 할 것
- 모르겠거나 정보가 부족하면 어디가 부족한지 얘기해서 요구할것.
- 커밋 메시지를 작성하기 위한 프로젝트임
- 파일명이 git_diff로 시작하는 파일은 export_diff.sh로 추출한 현재 작업한 내용이 담긴 파일임.
- 파일명이 git_log로 시작하는 파일은 export_recent_commits.sh로 추출한 최근 커밋 내역이 담인 파일임. 용도는 커밋 메시지를 작성할 때 비슷한 형식으로 쓰기 위한 참조용임
- 이슈번호도 작성하되 별다른 언급이 없으면 최근 커밋의 이슈번호 사용하지만 따로 브랜치 없으면 그 땐 안 써도 됨
- "Git diff/log 추출용 shell 스크립트 추가" 이런 내용은 쓰지마 이걸 왜 써.
- 말 좀 쉽고 간결하게
- 커밋 메시지의 description은 너무 자세하게 말고 가능하면 한줄로 작성해.
- 특별히 쓸만한게 없으면 description은 쓰지 않아도 돼.
- "description에는 실제로 한 작업만 써라. '왜', '목적', '효과' 같은 설명은 절대 쓰지 마라"
- "description은 단순 사실만 나열해라. ~를 위해, ~하여 같은 이유나 목적 설명 금지"
- commit message, description따로 말고 하나의 코드 블럭에다가 대답해
- 확인되지 않는 파일이 있는 경우에는 프로젝트에 직접 접근해서 확인은 해도 되지만 그 어떤 수정도 일어나서는 안 됨
```
