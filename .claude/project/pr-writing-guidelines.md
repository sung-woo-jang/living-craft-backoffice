# PR 작성 지침서

## 🎯 기본 원칙

### 1. 사용자 관점에서 작성
- **눈에 보이는 기능**만 작성
- 사용자가 실제로 체험할 수 있는 것에 집중
- 내부 구현 세부사항은 제외

### 2. 처음부터 완성된 것처럼 작성
- **"리팩토링", "개선", "적용", "분리", "마이그레이션"** 등의 표현 금지
- **"구현", "설계", "기반", "연동"** 등의 표현 사용
- 마치 처음부터 체계적으로 설계하고 구현한 것처럼 작성

### 3. 기존 컴포넌트 활용 명시
- 기존에 있던 컴포넌트를 사용한 경우 **"연동"** 표현 사용
- 새로 만든 것처럼 보이지 않도록 주의

## ❌ 제외할 내용들

### 기술적/구조적 내용
- FSD 아키텍처 관련 언급
- CSS 모듈, 스타일링 구현 방식
- 컴포넌트 구조 설계
- API 레이어 분리
- 파일 구조 변경사항

### 리팩토링 관련 표현
- "리팩토링", "개선", "최적화"
- "마이그레이션", "분리", "적용"
- "구조 개선", "코드 정리"

### 내부 구현 세부사항
- React Query 설정
- 상태 관리 스토어 구조
- 훅 분리
- 유틸리티 함수

## ✅ 포함할 내용들

### 기능별 분류
1. **지도 관련**: 위치 표시, 마커, 클러스터링 등
2. **검색/필터**: 검색어, 정렬, 필터링 옵션
3. **목록/상세**: 업체 리스트, 상세 모달, 정보 표시
4. **장바구니**: 선택, 제거, 제한 처리
5. **권한/액션**: 버튼 기능, 권한 체크

### 표현 방식
- **"구현"**: 새로운 기능을 만든 경우
- **"연동"**: 기존 컴포넌트를 활용한 경우
- **"기능"**: 사용자가 사용할 수 있는 동작
- **"처리"**: 로직이나 조건 분기

## 📝 작성 체크리스트

### 작업 유형
- **신규 기능 추가**만 체크
- 리팩토링 체크박스는 해제

### 최종 체크리스트
- **모든 항목 체크 해제** 상태로 제출
- 리뷰어가 직접 확인하도록 함

### 내용 검토
1. 사용자가 볼 수 없는 내용 제거했는가?
2. 리팩토링 관련 표현 제거했는가?
3. 기존 컴포넌트 활용을 "연동"으로 표현했는가?
4. 처음부터 완성된 것처럼 작성했는가?
